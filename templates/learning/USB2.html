The dangers of USB

from the example displayed the best course of action would be to leave the USB. this is because they can contain malware and cotain malicious exploits that can comprimise a device and steal data.

But how?

A USB device can be exploited to mimic a peripheral device such as a keyboard or mouse. with a few scripts, the USB can mimic keystrokes on the device too install malware and disable services. Scripts can perform these keystrokes thousands of times faster than a person and is practicaly instant. meaning that even if you have it plugged in for a second, its probably too late.

Security measures

employees of an organisation may not be aware of this danger and plug in comprimised devices onto workplace devices. But there is a solution that allows you to defend from these situations.

Measures

  • Endpoint protection - adopt a anti-virus solution that scans USB outputs in realtime to detect malware and remove before they can cause damage
  • Keep operating system up to date - protect against the latest exploits that comprimised USB devices may exploit
  • Disable Unnecessary USB Ports - can be done through group policies to totally disable ports on the system
  • Implement Device Control Policies - only allow authorized USB devices and deny the use of unauthorized ones
